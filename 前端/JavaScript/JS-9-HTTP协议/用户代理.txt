用户代理（User Agent）
    是指发起请求的客户端应用程序或设备的标识。在 Web 环境中，用户代理通常指的是 Web 浏览器，
    它负责向服务器发送 HTTP 请求，并处理服务器返回的响应。


用户代理包含了一些关键的信息，用于描述客户端应用程序或设备的特征，如：
    （1）浏览器名称和版本：用户代理可以提供浏览器的名称和版本信息，
        例如 "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/91.0.4472.124 Safari/537.36"。
    （2）操作系统信息：用户代理还可以包含客户端操作系统的相关信息，
        如操作系统名称、版本等。
    （3）设备类型：有些用户代理可以识别客户端的设备类型，
        如手机、平板电脑、桌面电脑等。
    （4）支持的功能和特性：用户代理可以指示其支持的功能和特性，
        例如支持的编码格式、支持的脚本语言、支持的插件等。

服务器可以通过检查请求中的用户代理信息来确定客户端的特征和能力，以便根据需要提供不同的响应或行为。在前端开发中，
有时会根据用户代理信息来进行浏览器兼容性处理或提供定制化的内容。

【总而言之】
    用户代理是指发起请求的客户端应用程序或设备的标识，它提供了关于客户端特征和能力的信息，用于服务器根据需要做出响应。
